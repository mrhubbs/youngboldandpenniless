Acorn¶

class acorn.Acorn(**kwargs)¶

Bases: object

This defines a very flexible serialization between Python objects and XML.

xml_tag = None¶

Tag to use for XML element created from the object. to ‘person’.

acorn_content = {}¶

This dictionary defines what class attributes Acorn will load from/save to XML.

acorn_content should be defined in the following way:

class Example(Acorn):
    acorn_content = Acorn.parse_content({
        'attrib_name': {'type': str, ...},
        'other_attrib_name': ...,
        ...
    })
__init__(**kwargs)¶

It is optional to call this when instantiating inheriting classes. If called, it will create all attributes with default values (and set them to the defaults) and create any attributes of ‘children’ type and set them to empty lists. It will create any attributes of ‘child’ type.

It will also set any attributes to any values specified in kwargs. These values override the defaults. For any attributes with ‘options’ specified, no checking is done to ensure the values in kwargs are permissible.

classmethod register_src(src_name, src)¶

Registers the source src under the name src_name.

src_name
str, name of source
src
Must inherit from BaseAcornSource.

Note

If there is already a source by src_name, it will be replaced by src.

Note

This acts ‘globally’. That is, the source will be registered to Acorn and all classes that inherit from it.

classmethod unregister_src(src_name)¶

Unregisters the source associted with the name src_name.

src_name
str, name of source

Note

This fails gracefully (without exceptions) in the event there is no such source.

Note

This acts ‘globally’. That is, the source will be unregistered from Acorn and all classes that inherit from it.

classmethod add_hook(event, hook)¶

Add hook to the event named event.

event
str, name of event. Current options are ‘fromxml’ and ‘toxml’.
hook

A callable of the form:

def hook(event, event_cls, obj):
    ...

  • event - name of event

  • event_cls - the class of the object generating the event. If the hook call resulted from doing:

    Acorn.fromxml(...)

    then event_cls would be Acorn.

  • obj - the object associated with the hook. For ‘fromxml’, this is the Python object being created. For ‘toxml’, this is the xml.etree.ElementTree.Element being created.

Note

If this hook is already assigned to event, it will not be added again.

Note

Unlike register_src(), this acts ‘locally’. That is, it only acts on the class it is called with, not all classes inheriting from Acorn.

classmethod remove_hook(event, hook)¶

Remove hook from the event named event.

event
str, name of event. Current options are ‘fromxml’ and ‘toxml’.
hook
The hook callable.

Note

This fails gracefully (without exceptions) in the event there is no such hook.

Note

Unlike register_src(), this acts ‘locally’. That is, it only acts on the class it is called with, not all classes inheriting from Acorn.

classmethod parse_content(content)¶

Processes the content dict into a more usable form for Acorn’s internal code. This should be used to define acorn_content.

classmethod fromxml(xml_src)¶

Create and return a new object loaded from xml_src.

xml_src
Either xml.etree.ElementTree.Element or a path of type str. If the first, the object will be loaded from the element. If the second, the XML file at the path xml_src will be parsed and the object loaded from the root element.
toxml(xml_dest=None, write_kwargs={'pretty_print': True})¶

Convert the object to an XML element.

xml_dest
Either xml.etree.ElementTree.Element or a path of type str. If the first, the object, when converted to XML, will be placed as a sub-element of xml_dest. If the second, the XML will be written to the path.
write_kwargs
kwargs to pass to etree.ElementTree.write() if xml_dest is a path

Acorn’s Sources¶

The various source objects provide facilities to load/save data to/from various parts of an XML data structure (e.g. from an element’s attribute, text, or child element, etc.). They are created behind-the-scenes when Acorn parses content definitions and do the grunt work for the fromxml() and toxml() operations.

It is possible to create custom sources.

class acorn_base.BaseAcornSource(meta)¶

The base class for every source.

class acorn_base.AcornTextSource(meta)¶

Source to get data from element’s text.

fromxml(name, obj, xml_el)¶

Loads an object’s attribute from the XML element’s attribute or child’s texts.

This applies options and default.

class acorn_base.AcornAttrSource(meta)¶

Source to get data from element’s attribute.

class acorn_base.AcornSubTextSource(meta)¶

Source to get data from element’s child’s text.

class acorn_base.AcornChildSource(meta)¶

Source to load object from element’s child.

class acorn_base.AcornChildrenSource(meta)¶

Load a series of objects from the direct children (children’s children are ignored).

class Weapon(Acorn):
    ...

class Person(Acorn):
    xml_tag = 'person'
    acorn_content = Acorn.parse_content({
        ...
        'weapons': {'type': Weapon, 'src': 'children'}
    })

person = Person.fromxml(...)
print(person.weapons)
